# SUCCESS_CRITERIA

A v1.1.1 fica validada quando:
- houver mais do que 1 fixture no snapshot
- os runs continuarem green ou controlados
- o snapshot mantiver forma consistente
- o consumo pelo Orchestrator se mantiver estável
- os logs continuarem legíveis e curtos

Estados finais:
- green: objetivo cumprido
- red_coverage: bootstrap ok, mas cobertura insuficiente
- red_bootstrap: falha antes da calibração funcional
