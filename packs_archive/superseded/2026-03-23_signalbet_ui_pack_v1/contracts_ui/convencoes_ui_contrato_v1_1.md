# Convenções UI — Contrato v1.1

## objetivo
Assegurar que a UI representa estados e campos compatíveis com a arquitetura comum.

## convenções de leitura
- `global_score` = força agregada da oportunidade
- `confidence_norm` = confiança normalizada
- `edge_norm` = edge normalizado
- `risk_norm` = risco normalizado
- `priority_tier` = classe de prioridade
- `eligibility` = elegibilidade operacional
- `decision_status` = estado da decisão da banca
- `execution_status` = estado no pipeline operacional
- `data_quality_flag` = qualidade/confiança dos dados

## princípio
Score analítico nunca deve ser visualmente confundido com decisão final.
